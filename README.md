# MedMitra — AI Prescription Safety System

MedMitra is a full-stack web application that helps patients and caregivers analyze prescriptions for drug interactions, duplicate therapies, overdose risks, and other clinical safety concerns. Upload one or more prescription images, and the AI extracts medications, runs a safety analysis, and delivers the results — including an optional WhatsApp summary.

## Features

- **Prescription OCR** — Upload `.png`, `.jpg`, `.webp`, or `.json` prescription files. Gemini Vision extracts medication names, dosages, frequencies, and durations from the image.
- **Multi-Prescription Analysis** — Upload prescription images and then it will combine and analyse all the medications together.
- **AI Safety Analysis** — Google Gemini (`gemini-2.5-flash`) acts as a senior clinical pharmacist and checks for drug interactions (pairwise and 3-drug combos), duplicate therapy, class duplicates, overdose risks, and route-of-administration warnings.
- **Risk Scoring** — Every analysis produces a risk score (0–10) labeled MODERATE / HIGH based on the number and severity of clinical alerts.
- **Results Dashboard** — Displays medications, safety alerts (high / moderate), instructions derived from clinical alerts, and a summary card.
- **WhatsApp Delivery** — Send the full analysis summary to any WhatsApp number via Twilio. Messages are split automatically if they exceed the character limit.
- **Dark / Light Mode** — Theme toggle in the header powered by `next-themes`.
- **Features Page** — A dedicated page describing the core capabilities of the app.

## Getting Started

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [Google AI Studio](https://aistudio.google.com/) API key (`GEMINI_API_KEY`)
- A [Twilio](https://www.twilio.com/) account for WhatsApp (optional)

---

### Backend Setup

```bash
cd backend

# Create and activate virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS / Linux

# Install dependencies
pip install -r requirements.txt

# Create .env file
cp .env.example .env
# Fill in your keys (see Environment Variables section below)

# Start the server
uvicorn app.main:app --reload
```

The API will be available at `http://127.0.0.1:8000`.

---

### Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Start the development server
npm run dev
```

The frontend will be available at `http://localhost:3000`.

## Environment Variables

Create a `.env` file inside the `backend/` directory:

```env
# Required — Google Gemini API key
GEMINI_API_KEY=your_gemini_api_key_here

# Optional — Twilio credentials for WhatsApp delivery
TWILIO_SID=your_twilio_account_sid
TWILIO_AUTH=your_twilio_auth_token
TWILIO_FROM=whatsapp:+14155238886
```

> **WhatsApp note:** The recipient must first join the Twilio Sandbox by sending `join house-guard` to `+14155238886` on WhatsApp before they can receive messages.

## How It Works

1. **Upload** — The user drags and drops one or more prescription images (PNG, JPG, WEBP) or a JSON file onto the upload card.
2. **OCR** — The backend sends each image to Gemini Vision (`ocr_service.py`), which extracts a structured list of medications including name, dosage, frequency, duration, and notes.
3. **Analysis** — All extracted medications are passed to `analysis_service.py`, which sends them to Gemini with a clinical pharmacist system prompt. Gemini returns a structured JSON object with a risk score, clinical alerts, and a medication summary.
4. **Display** — The frontend maps the backend response into the Results Dashboard showing medications, safety alerts, and actionable instructions.
5. **WhatsApp (optional)** — The user enters their phone number and clicks Send. The backend formats the analysis into readable WhatsApp messages and delivers them via Twilio.
